## Commands

```bash
bun dev              # Start Convex + Next.js dev servers concurrently
bun run build        # Production Next.js build
bun run import       # Import data from Supabase PG into Convex (wipes + reimports)
bunx convex run <module>:<function>  # Run any Convex function
```

Use `bun`/`bunx` — never `npm`/`npx`.

## Architecture

**Next.js 16.2 + Convex** — No API routes. All backend logic lives in `convex/` as Convex queries and mutations. The frontend is entirely client-rendered pages using Convex React hooks. Data is imported from a Supabase PostgreSQL database via `scripts/import-pg.ts`.

### Backend (`convex/`)

- **schema.ts** — 8 tables: `orgs`, `people`, `funds`, `projects`, `grants`, `personOrgs`, `fundPersons`, `personProjects`
- **orgs.ts** — `listOrgs`, `getBySlug`
- **funds.ts** — `list`, `getBySlug`, `getPeople`, `getGrants`
- **people.ts** — `getByOrg` (via personOrgs join)
- **projects.ts** — `getByOrg`
- **grants.ts** — `getByRecipientOrg`, `getByFunderOrg` (resolves funder/recipient names)
- **import.ts** — Mutations for bulk import: `clearAll`, `insertOrgs`, `insertPeople`, `insertFunds`, `insertProjects`, `insertGrants`, `insertPersonOrgs`, `insertFundPersons`, `insertPersonProjects`
- **`_generated/`** — Auto-generated types; never edit. Regenerated by `bunx convex dev`.

### Frontend (`app/`)

- **layout.tsx** — Server component. Fonts (Vollkorn SC, Jost, JetBrains Mono), header with nav, wraps children in `ConvexClientProvider`.
- **page.tsx** — Client component. Sortable/filterable org table. Links to `/org/[slug]`.
- **org/[slug]/page.tsx** — Client component. Org detail with stats, people, projects, grants received/given.
- **funds/page.tsx** — Client component. Fund cards with key stats.
- **funds/[slug]/page.tsx** — Client component. Fund detail with metadata, people, searchable/sortable grants table.
- **convex-client-provider.tsx** — Client component wrapping `ConvexProvider`.

### Key Patterns

- All pages are `"use client"` — they use Convex React hooks (`useQuery`, `useMutation`).
- `params` is a `Promise` in Next.js 16 — use `use(params)` in client components.
- Conditional queries use the `"skip"` pattern: `useQuery(api.x.y, condition ? args : "skip")`.
- Loading: `undefined` = loading, `null` = not found.

## Design System

Tailwind v4 with `@theme inline` in `globals.css`. Custom warm color palette (warm-50 through warm-900), accent rose (#A4325A). Fonts: Vollkorn SC (logo/headings), Jost (body), JetBrains Mono (numbers/mono).
