## Commands

```bash
bun dev              # Start Convex + Next.js dev servers concurrently
bun run build        # Production Next.js build
bun run seed         # Run seed mutation: convex run seed:seed
bunx convex run <module>:<function>  # Run any Convex function (e.g. seedMetr:seedMetr)
```

Use `bun`/`bunx` — never `npm`/`npx`.

## Architecture

**Next.js 16.2 + Convex** — No API routes. All backend logic lives in `convex/` as Convex queries and mutations. The frontend is entirely client-rendered pages using Convex React hooks.

### Backend (`convex/`)

- **schema.ts** — Three tables: `orgs` (indexed by slug), `staff` (indexed by orgId), `comments` (indexed by orgId)
- **orgs.ts** — `listOrgs` (all), `getOrgBySlug` (by slug index, `.unique()`)
- **staff.ts** — `getStaffByOrg` (by orgId index)
- **comments.ts** — `getCommentsByOrg` (query), `addComment` (mutation, uses `Date.now()`)
- **seed.ts / seedMetr.ts** — Data seeding mutations
- **`_generated/`** — Auto-generated types; never edit. Regenerated by `bunx convex dev`.

### Frontend (`app/`)

- **layout.tsx** — Server component. Fonts (Vollkorn SC, Jost, JetBrains Mono), header, wraps children in `ConvexClientProvider`.
- **page.tsx** — Client component. Sortable/filterable org table. Links to `/org/[slug]`.
- **org/[slug]/page.tsx** — Client component. Org detail with stats, staff, comments, review form.
- **convex-client-provider.tsx** — Client component wrapping `ConvexProvider`.

### Key Patterns

- All pages are `"use client"` — they use Convex React hooks (`useQuery`, `useMutation`).
- `params` is a `Promise` in Next.js 16 — use `use(params)` in client components.
- Conditional queries use the `"skip"` pattern: `useQuery(api.x.y, condition ? args : "skip")`.
- Loading: `undefined` = loading, `null` = not found.

## Design System

Tailwind v4 with `@theme inline` in `globals.css`. Custom warm color palette (warm-50 through warm-900), accent rose (#A4325A). Fonts: Vollkorn SC (logo/headings), Jost (body), JetBrains Mono (numbers/mono).
